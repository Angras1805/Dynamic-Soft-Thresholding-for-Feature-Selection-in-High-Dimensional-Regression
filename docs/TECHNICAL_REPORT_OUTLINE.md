## 1. Problem Statement

- Predict house prices with many correlated predictors (after one-hot encoding).
- Goal: **feature selection** (sparse coefficients) without losing accuracy.

## 2. Dataset and Preprocessing

- Dataset: **Kaggle House Prices: Advanced Regression Techniques**
- Target: `SalePrice` (use \(\\log(1+\\text{SalePrice})\) for stability).
- Preprocessing (same for all methods):
  - Numeric: median impute + standardize
  - Categorical: most-frequent impute + one-hot encoding
- Report feature counts from `outputs/preprocess_info.json`.

## 3. Mathematical Formulation

### 3.1 Objective

\[
\min_{\\beta\\in\\mathbb{R}^p} \\frac{1}{2n}\\|X\\beta-y\\|_2^2 + \\lambda\\|\\beta\\|_1
\]

### 3.2 Proximal Gradient and Soft-Thresholding

- Prox step of \(\\|\\beta\\|_1\) is the soft-threshold operator.

### 3.3 Dynamic Soft-Thresholding (Proposed)

- Coordinate-wise adaptive thresholds:
  - \(g_i^k = \\frac{|\\beta_i^k|}{|\\beta_i^k|+\\epsilon}\)
  - \(s(k)=1 + \\frac{A}{(1+k)^p}\)
  - \(\\tau_i^k = \\alpha\\lambda(\\gamma_0 + \\gamma_1(1-g_i^k))s(k)\)
- Interpretation:
  - Near-zero coefficients get **bigger** thresholds early → pruning
  - Confident coefficients get smaller relative shrinkage later → fine-tuning in correlated groups

## 4. Baselines

- Ridge regression (L2 penalty)
- Standard LASSO (static L1 penalty)

## 5. Experimental Protocol

- Cross-validation with fixed seed and identical folds across methods.
- Hyperparameter tuning:
  - Ridge: grid of \(\\alpha\\)
  - LASSO: grid of \(\\lambda\\)
  - Dynamic LASSO: grid over \(\\lambda\\) and \(\\gamma_1\) (keep \(\\gamma_0\) fixed)
- Metrics:
  - Accuracy: MSE
  - Sparsity: number of nonzero coefficients
  - Efficiency: mean runtime per fold
